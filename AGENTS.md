# Operating protocols for agents working on poki soweli

Companion to `CLAUDE.md`. CLAUDE.md is the quick-orient entry point; this file is the extended reference for how work happens here.

## Docs → Tests → Code (the one rule that matters most)

Strict dependency chain:

1. **Docs** describe what the game must be — product vision, acceptance criteria, slice goals.
2. **Tests** describe what the code must do to satisfy the docs — referenced by doc section or phrase.
3. **Code** satisfies docs and tests — the implementation of the contract, not the source of truth.

When tests fail, the question is "is the code wrong or is the test wrong?" — judged against the doc, not against the other. If a test contradicts the doc, the test is wrong. If the doc is silent on what the test asserts, the doc changes first, then the test, then the code.

When adding a feature, the first question is "what does the doc say this should do?" — if no doc answers, write the doc section before thinking about implementation.

## The content pipeline is load-bearing

Every piece of game content flows through this pipeline:

```
src/content/spine/<kind>/<id>.json   (hand-authored, EN strings tagged for TP resolution)
   ↓ scripts/build-spine.mjs
   ↓   validates Zod schemas
   ↓   resolves EN → canonical TP via src/content/corpus/tatoeba.json
   ↓ src/content/generated/world.json   (committed for reproducibility)
   ↓ src/game/content/loader.ts
   ↓ Phaser scenes + Solid overlays
```

Authors edit `spine/`, never `generated/`. `pnpm prebuild` runs the full pipeline and gates `pnpm build`. CI enforces it on every PR via `.github/workflows/content-validate.yml`.

## The asset pipeline (NEW this PR — preserved against the Godot regression)

Every sprite in the game comes from one of these trees. Mixing outside the allowlist introduces the tonal inconsistency that sank the previous playthrough.

| Category | Source | Where |
|---|---|---|
| Tilesets | Fan-tasy family (6 biome packs) | `public/assets/tilesets/{core,seasons,snow,desert,fortress,indoor}/` |

## Testing strategy

**Docs** → `docs/SLICE_CHECK.md` is the acceptance-criteria manifest: one bullet per observable behavior the game must exhibit.

**Tests** → `tests/` contains a Vitest browser harness (Playwright-backed) that replays a deterministic "optimal playthrough" playbook through the real Phaser game. The playbook is auto-generated by a factory from `world.json`. Each scene forks the replay to exercise every meaningful interaction (dialog branches, combat actions, item use), capturing named screenshots at each step.

See `docs/TESTING.md` for the harness spec.

**Code** → satisfies both. Run `pnpm test` locally for every code change. CI runs it on every PR.

## Languages

- **TypeScript strict** for all game code.
- **GDScript** is Godot-specific and **not used** in this branch. If you see a `.gd` file in git history, it's from the Godot-era `main` — ignore it. This spike descends from the pre-Godot tip.

- **Tiled `.tmx`/`.tsx`** for tilemaps and tilesets — but `.tmx` under `src/tiled/` and `.tmj` under `public/assets/maps/` are **emitted artifacts**, never hand-edited. Source-of-truth specs live in `scripts/map-authoring/specs/<id>.ts`.

## When something fights you

- **Tiled path refs break after a move** — `.tsx` uses `../../Art/...`, `.tmx` uses `../Tilesets/...`. Preserve the pack's internal `Art/` + `Tiled/` layout when reorganizing.
- **`pnpm author:verify` fails** — the .tmx on disk drifted from the spec (or is orphaned, or missing). DO NOT edit the `.tmx` to fix it; edit the spec under `scripts/map-authoring/specs/` and re-run `pnpm author:build <id>`. The drift message tells you which spec to touch. Hand-edited `.tmx` is the error; regenerating is the fix.
- **`pnpm validate-tp` rejects a line** — rewrite the EN, never hand-author TP. See `docs/WRITING_RULES.md`.
- **Phaser spritesheet frames look wrong** — check the filename. Grid sizes are encoded: `axe_warrior_16x16.png` = `frameWidth: 16, frameHeight: 16`.
- **Dev server shows blank canvas** — WebGL context may have dropped; see `src/game/PhaserGame.tsx` for the recovery path (already wired).

## What's unique about this game (design constraints)

- **Player has no stats.** The party of ≤ 6 creatures is the character sheet.
- **Five creature types**: seli (fire) / telo (water) / kasi (plant) / lete (ice) / wawa (strong). Rock-paper-scissors with wawa as the neutral bruiser.
- **No translation UI.** Vocabulary is picked up diegetically through play. The player never sees an English gloss dictionary.
- **Seven regions → seven jan lawa → one final boss (green dragon).** The only creature with a dedicated death animation.
- **Kid audience.** "Dread knight" > "death knight". Tone is fierce-but-friendly.
