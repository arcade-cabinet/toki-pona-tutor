## Process notes for future batches

- **Branch protection is permissive for docs-only PRs.** PR #98 auto-merged with a failing unit test. Not something to rely on — fix tests before merging, always.
- **The drift-guard test (`docs-current-state.test.ts > keeps code-spanned docs/*.md references backed by repo files`)** is strict. Either exclude `docs/plans/` before adding forward-looking PRDs, or use prose refs for docs that don't exist yet.
- **Auto-merge across many coupled docs PRs churns main.** If you open N docs PRs that each reference the others, main's drift guard tests flip red each time one merges. Sequence them or bundle them.
- **The OAuth token in this session** cannot enable auto-merge on workflow-touching PRs. Keep those in a separate workflow-scoped actor's queue.
